🤖 What is StandarCloud?

StandarCloud is a central repository for all the meta-files that AI coding assistants use to understand your projects. Instead of re-writing the same instructions in every repository, you define them once here and let the StandarCloud MCP server inject them automatically into any AI tool that supports the Model Context Protocol (MCP).

Supported meta-files per AI tool
ToolMeta-file(s)Purpose
🟣 Claude / Claude Code CLAUDE.md Project rules, commands and conventions injected into every session
🔵 VS Code Copilot .github/copilot-instructions.md Workspace-level custom instructions for GitHub Copilot
🟡 Cursor .cursorrules · .cursor/rules/*.mdc Global and per-directory coding rules for the Cursor AI editor
🟢 Windsurf .windsurfrules Project-level coding instructions for Codeium Windsurf
⚪ OpenCode / Aider AGENTS.md · .aider.conf.yml Agent behaviour, conventions and hints for agentic coding tools
🔴 Cline / Roo-Cline .clinerules · .roorules Custom tool rules and memory injected by these VSCode extensions
🟠 Continue.dev .continue/config.json Model selection, context providers and slash-command definitions
🔵 Devin DEVIN.md Project setup, conventions and secrets guidance for Devin sessions
⚫ Any MCP-enabled tool AGENTS.md Standardised agent instruction file (OpenAI Agents SDK convention)
🔄 Backup & Sync flow

The MCP client reads your local AI meta-files and pushes them to StandarCloud. From there they can be pulled into any other machine or project.

☁️ StandarCloud API

↕
🗄️ Asset store
versioned per project
  1. Push (backup) – The MCP tool reads every AI meta-file in your repo root and uploads them as assets linked to your project slug.
  2. Store – Each asset is stored with its filename, content and a timestamp. Previous versions are kept for auditing.
  3. Pull (sync) – On a new machine or in CI, the MCP tool fetches all assets for a project and writes the files back to disk — your AI tools pick them up automatically.
  4. Share – Team members using the same project slug share identical instructions, ensuring consistent AI behaviour across the whole team.
🚀 Install the StandarCloud MCP server
1
Generate an API Token

Go to API Tokens, create a token for your project and copy it. This token authenticates every MCP request.

2
Install the MCP package
pip install standarcloud-mcp
# or run directly without a global install (requires uv)
uvx standarcloud-mcp --help
3
Register in your AI tool

Add to ~/.claude/claude_desktop_config.json or register via CLI: 

claude mcp add standarcloud \
  -e STANDARCLOUD_API_URL=http://localhost:8000 \
  -e STANDARCLOUD_API_TOKEN=YOUR_TOKEN \
  -- python -m standarcloud_mcp

MCP tools exposed: sync_assets, pull_assets, list_projects. Ask Claude to "sync my AI files to StandarCloud".

Create/edit .cursor/mcp.json in your project root:

{
  "mcpServers": {
    "standarcloud": {
      "command": "python",
      "args": ["-m", "standarcloud_mcp"],
      "env": {
        "STANDARCLOUD_API_URL": "http://localhost:8000",
        "STANDARCLOUD_API_TOKEN": "YOUR_TOKEN"
      }
    }
  }
}

Add to .vscode/mcp.json (workspace) or to user settings.json: 

{
  "servers": {
    "standarcloud": {
      "type": "stdio",
      "command": "python",

Use the MCP tools

Once registered, your AI assistant gains these tools automatically:

ToolWhat it does

📡 REST API quick reference

All endpoints are under /api/v1/ and require the header Authorization: Token YOUR_TOKEN.

MethodEndpointDescription
GET/api/v1/projects/List projects
POST/api/v1/projects/Create project
GET/api/v1/assets/?project=List assets for a project
POST/api/v1/assets/sync/Upsert (push) assets
GET/api/v1/assets/pull/?project=Pull assets to disk
