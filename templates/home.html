✅ Estado Actual (Web + MCP)

StandarCloud centraliza contexto en la web (projects, assets, sessions, memory entries) y lo proyecta localmente mediante MCP. El snapshot cloud es canonico y se sincroniza al workspace por entorno.

AreaFuncionalidad vigenteEstado
Web / Projects Listado, detalle, editar, borrar y navegacion de assets por proyecto Activo
Web / Assets Tipos: skill, agent, prompt, spec, other. Acciones en lista: View, Edit, Delete Activo
Web / Asset Form Selectores de Environment (.github, .opencode, .claude) y Context (agents/prompts/skills/specs/assets) con path autogenerado Activo
Cloud Sync ai_cloud_sync (web to workspace mirror, push opcional), ai_cloud_push, ai_cloud_status Activo
Session Memory ai_session_ensure, ai_memory_add, ai_memory_list, ai_memory_materialize_history Activo

🚀 MCP: instalacion y uso real
1
Genera token API

Crea un token en API Tokens. Se usa para autenticar el MCP contra la API web.

2
Configura variables de entorno
export AI_CONTEXT_MANAGER_BASE_URL="http://localhost:8000"
      export AI_CONTEXT_MANAGER_TOKEN="YOUR_TOKEN"
3
Arranca el servidor MCP
cd mcp-standard-ai
python -m mcp_server
4
Tools MCP disponibles

Si hay token/base_url configurados, se habilitan herramientas cloud y de memoria.

ToolWhat it does
ai_cloud_statusEstado local vs cloud (synced, only_local, only_cloud)
ai_cloud_syncMirror cloud to workspace (push opcional)
ai_cloud_pushPublica cambios locales hacia la web
ai_cloud_pullDescarga assets remotos al workspace (deprecated, use sync)
ai_session_ensureCrea/reutiliza sesiones vinculadas a proyecto
ai_memory_addGuarda observaciones/decisiones en memoria cloud
ai_memory_listLista memoria por proyecto/sesion/tag
ai_memory_materialize_historyMaterializa historico en formato nativo (Copilot/Claude/OpenCode)
📡 API REST actual (Django)

Base: /api/v1/. Autenticacion por token: Authorization: Token YOUR_TOKEN.

MethodEndpointDescription
GET/api/v1/projects/Lista proyectos del usuario
POST/api/v1/projects/Crea proyecto
GET/api/v1/assets/?project=<slug>&type=<tipo>Filtra assets por proyecto y tipo
POST/api/v1/assets/Crea asset
GET/api/v1/sessions/?project=<slug>Lista sesiones de memoria
POST/api/v1/sessions/Crea sesion
GET/api/v1/sessions/<id>/entries/Lista entradas de una sesion
POST/api/v1/sessions/<id>/entries/Agrega entrada de memoria
GET/api/v1/memory-entries/?project=<slug>&tag=<tag>Consulta global de memoria
🔄 Flujo recomendado de trabajo
  1. Crea/edita assets desde la web usando Environment + Context para generar paths consistentes.
  2. En tu workspace, ejecuta ai_cloud_sync para proyectar cloud a local (modo espejo).
  3. Si hiciste cambios locales, corre ai_cloud_sync con push: true o ai_cloud_push.
  4. Valida estado final con ai_cloud_status y revisa .acm/sync-state.json.
